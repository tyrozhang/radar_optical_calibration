# 雷达光电联动标定 - 关键概念备忘录

> 本文档针对本人在理解 CalibrateV2 标定逻辑时的核心疑惑点进行固化，防止后续重复迷惑。

---

## 一、核心结论（先记住）

**标定分两步，计算独立、补偿串联：**
1. **光电标定**（光电设备实测时的误差）：先求出光电自身的机械零偏（ΔAz0 / ΔEl0），把光电变成"可信基准"。
2. **雷达标定**：以光电站位置为原点做纯几何计算，也就是在站心坐标系中，雷达上报坐标对应目标真实坐标的系统性角度偏差（dAz_radar / dEl_radar）。

**使用时必须两步串联补偿**：雷达上报坐标 → 转光电视角 → 扣雷达补偿 → 扣光电零偏 → 精确指向。

---

## 二、为什么不能只标雷达？

因为**光电自身工作时也有误差**。

光电转台的编码器零点、光轴安装误差会导致它测出来的角度自带一个固定偏移（ΔAz0 / ΔEl0）。如果不先标出来：
- 你无法知道"真值角度"到底是多少；
- 后续用光电去评估雷达时，基准本身就是歪的；
- 最终光电指向时，也会自带这个固定偏移。

 **注意：**

 **标定计算阶段**：光电标定与雷达标定相互独立，没有数据依赖，可并行执行。
 **系统方法论与运行阶段**：光电标定是角度基准可信的前提，两者补偿必须串联使用，缺一不可。

---

## 三、第一步：光电标定（CalibrateOptical）

### 3.1 标的是什么？
光电设备的**静态零点偏差**：
- `ΔAz0`：方位零偏（光电实测方位 − 几何方位）
- `ΔEl0`：俯仰零偏（光电实测俯仰 − 几何俯仰）

### 3.2 怎么算的？

通过光电站看真实目标的理想角度和实测观察角度之差来进行计算。

| 输入 | 处理 | 输出 |
|------|------|------|
| 光电站 BLH + 目标 RTK 真值 BLH | `blhToAzEl(target, optical)` → **geoAz / geoEl**（理想几何角） | 每个点的 `dAz0_i = azMeasured − geoAz`，`dEl0_i = elMeasured − geoEl` |
| 所有点的 `dAz0_i` / `dEl0_i` | 按距离倒数加权 + **3σ 野值剔除** + 加权平均 | `ΔAz0`、`ΔEl0`、RMS |

### 3.3 geoAz / geoEl 到底是什么？
**纯几何计算结果**，与光电设备无关。

`geoAz` = 从光电站看向目标真值点的**理想方位角**  
`geoEl` = 从光电站看向目标真值点的**理想俯仰角**

它们是拿两组 BLH 坐标（目标真值 + 光电站）做纯数学转换算出来的，**不是光电测出来的**，只是用来跟光电实测角对比的参照系。

## 四、第二步：雷达标定（RadarErrorAnalyzer）

### 4.1 标的是什么？
雷达上报坐标的**系统性角度偏差**（在以光电站为原点的站心坐标系下）。

### 4.2 怎么算的？
**关键：全程不用光电实测角，只用光电站的"位置坐标"作为几何原点。**

| 步骤 | 计算 | 含义 |
|------|------|------|
| ① 雷达视角 | `blhToAzEl(radarBlh, opticalBlh)` | 从光电站看，**雷达上报坐标**对应什么角度 |
| ② 真值视角 | `blhToAzEl(targetBlh, opticalBlh)` | 从光电站看，**真值目标**对应什么角度 |
| ③ 求差 | `azError = radar视角.az − 真值视角.az` | 雷达上报坐标相对真值的角度偏差 |

`dAz_radar` / `dEl_radar` 就是上述 `azError` / `elError` 的统计均值。

### 4.3 按距离分段的意义
雷达误差往往随距离变化（波束展宽、多径等）。程序将样本按**雷达到目标的 3D 斜距**分段统计，若各段差异显著则输出**分段补偿**，否则输出**固定补偿**。

---

## 五、关键概念澄清（防止再迷惑）

### 5.1 "光电"在雷达标定中到底指什么？
- **不是**指光电设备的测量能力（`azMeasured` / `elMeasured`）。
- **是**指光电站精确已知的**安装位置坐标**（`opticalBlh`）。

雷达标定中光电设备"只提供了屁股坐在哪里的坐标"，它的"眼睛测得准不准"完全没参与。

### 5.2 雷达标定过程中光电零偏被消除了吗？
**没有被消除，而是根本没有被引入。**

因为 `blhToAzEl` 是纯几何公式（输入两个坐标点，输出相对方位角），不涉及任何设备测量值。既然光电测量值根本没进场，它的零偏自然不会影响结果。

### 5.3 雷达补偿量 dAz_radar 是在哪个坐标系下定义的？
**光电视角域**（以光电站为原点的站心坐标系）。

所以最自然的用法是：先把雷达上报 BLH 转到光电视角，再直接加减这个补偿量。**不需要、也不建议**先把补偿反推回 BLH 坐标域。

### 5.4 补偿顺序（先转角度，再两级角度补偿）

```
雷达上报 BLH
    │
    ▼
blhToAzEl(雷达BLH, 光电站BLH)
    │
    ▼
┌─────────────────────────────────────┐
│      光电视角下的雷达角度            │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  应用雷达补偿                       │
│  - dAz_radar / dEl_radar            │
│  （按距离选段，减去雷达系统误差）    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  应用光电补偿                       │
│  - ΔAz0 / ΔEl0                      │
│  （减去光电自身零偏）                │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         光电精确指向角度             │
└─────────────────────────────────────┘
```


## 六、符号约定

| 符号 | 含义 | 备注 |
|------|------|------|
| `ΔAz0` / `ΔEl0` | 光电零点偏差 | 光电实测角 − 几何角 |
| `dAz_radar` / `dEl_radar` | 雷达上报坐标的角度偏差 | 雷达视角.az − 真值视角.az |
| `azError > 0` | 雷达上报比真值偏**右** | 即雷达实际偏**左** |
| `elError > 0` | 雷达上报比真值偏**下** | 即雷达实际偏**上**（俯视方向） |
| 角度归一化 | `[-180, 180)` | 方位角做归一化，俯仰角不做 |

## 七、常见误区速查表

| 误区 | 正解 |
|------|------|
| "雷达标定用了光电测量值，所以受光电误差影响" | 雷达标定全程没用 `azMeasured` / `elMeasured`，只用 `opticalBlh` 坐标做几何计算 |
| "只标定雷达就够了，光电不用标" | 光电自身有机械零偏，不标则基准不准、最终指向也不准 |
| "雷达补偿应该先在 BLH 坐标域扣除" | 雷达补偿量是角度偏差，最自然的用法是在角度域扣除 |
| "blhToAzEl 输出的是光电测出来的角度" | `blhToAzEl` 输出的是**纯几何角度**，与任何设备测量无关 |
| "geoAz/geoEl 包含了光电误差" | `geoAz`/`geoEl` 是拿 RTK 真值和光电站坐标算的理想角，与光电设备无关 |

---

## 八、一句话串起来

> **先用 RTK 真值把光电自己的零偏标出来，再把光电站当几何原点，对比"雷达上报坐标的角度"和"真值坐标的角度"，得到雷达的角度偏差。使用时把雷达坐标转到光电视角，先扣雷达偏差再扣光电零偏，光电就能精确指向真实目标。**
